# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**Purposemed.ai** executive AI training documentation site for PurposeMed, built with **Docusaurus 3.8.1**. Deployed to `https://ximxim.github.io/WorkflowyOS-Docs/` via GitHub Pages. Dark-mode only, docs-only mode (docs served at root `/`). Brand name in UI is "Purposemed.ai"; the footer attribution reads "With ❤️ from Workflowy.ai".

The content covers 10 AI training domains tailored for healthcare executives: prompting, context management, research, content creation, workflow automation, MCP connections, AI agents, governance/compliance (PIPEDA/PHIPA/HIPAA), ROI measurement, and change management.

**PurposeMed context**: Cross-border telehealth company with three service lines — Freddie (HIV/PrEP), Frida (ADHD), Foria (gender-affirming care). Content should always reference these when giving healthcare examples.

### Key Configuration Decisions

- **Docs at root**: `routeBasePath: '/'` — no separate landing page
- **Dark mode forced**: `disableSwitch: true`, `respectPrefersColorScheme: false`
- **Breadcrumbs hidden**: `breadcrumbs: false`
- **Broken links throw**: `onBrokenLinks: 'throw'` — build fails on broken links
- **Mermaid diagrams**: `markdown.mermaid: true` + `@docusaurus/theme-mermaid` theme
- **LLM plugin**: `docusaurus-plugin-llms` generates `llms.txt` and `llms-full.txt` at build time
- **Local search**: `@easyops-cn/docusaurus-search-local` (no Algolia)
- **Content max-width**: 800px via CSS `.main-wrapper main`
- **Blog disabled**: Training site only
- **GitHub Pages baseUrl**: `/WorkflowyOS-Docs/`

## Content Guidelines

- **Prefer markdown over HTML** — use markdown headers, images, lists, emphasis, links, code blocks
- Use Docusaurus admonitions (:::note, :::tip, :::warning, :::danger, :::info)
- Kebab-case filenames
- Images go in `static/img/` with relative paths
- Internal links use relative paths
- Add `sidebar_position` in frontmatter to control ordering
- Training/curriculum tone, second-person voice addressing executives
- Healthcare examples throughout, grounded in PurposeMed context (Freddie/Frida/Foria)

## Mermaid Diagrams

The site uses Mermaid diagrams extensively (39 diagrams across 24 pages). When adding or editing diagrams:

- Use ` ```mermaid ` code fences in markdown files
- **Quote all node labels** containing special characters: `A["Label with (parens) or commas"]`
- Never use bare parentheses in labels — Mermaid interprets them as node shapes
- Keep diagrams concise: 5–15 nodes max for readability
- Preferred types: `flowchart TD/LR` (most common), `timeline`, `sequenceDiagram`, `mindmap`, `quadrantChart`
- Dark mode renders automatically via Docusaurus theme-mermaid
- Always run `npm run build` after editing diagrams to verify syntax

## Sidebar Pattern

The sidebar uses HTML headings for section labels + `autogenerated` items from directories. Do NOT wrap autogenerated items in a `category` when the category would be the only child under a section heading — this creates redundant nesting. Place `autogenerated` items directly under the HTML heading.

```typescript
// CORRECT — flat structure
{ type: 'html', value: 'SECTION NAME', className: 'sidebar-category-heading' },
{ type: 'autogenerated', dirName: 'section-dir' },

// WRONG — redundant nesting
{ type: 'html', value: 'SECTION NAME', className: 'sidebar-category-heading' },
{ type: 'category', label: 'Category', items: [{ type: 'autogenerated', dirName: 'section-dir' }] },
```

## Learned Rules

### 2026-02-28 — FAQStructuredData prop is `faqs`, not `items`
**Issue**: Build failed with SSG error because `<FAQStructuredData items={...}>` was used
**Rule**: The FAQStructuredData component expects a `faqs` prop (array of `{question, answer}`). Using `items` causes a runtime `.map()` error during static site generation.

### 2026-02-28 — Autogenerated category indexes are not direct routes
**Issue**: 7 pages linked to `/governance` which doesn't resolve as a standalone page
**Rule**: When sidebar uses `type: 'autogenerated'`, the generated category index page is NOT a routable URL like `/category-name`. Always link to a specific page within the category (e.g., `/governance/patient-data-and-compliance`), not the category itself.

### 2026-02-28 — Docusaurus theme packages must match core version
**Issue**: `@docusaurus/theme-mermaid@3.9.2` was installed but core was 3.8.1, causing build failure
**Rule**: All `@docusaurus/*` packages must have the exact same version as `@docusaurus/core`. Always install with explicit version: `npm install --save @docusaurus/theme-mermaid@3.8.1`.

### 2026-02-28 — Mermaid node labels need quotes for special characters
**Issue**: Bare parentheses in node labels caused Mermaid parse errors
**Rule**: Always wrap Mermaid node labels in double quotes when they contain parentheses, colons, slashes, commas, or ampersands: `A["Label with (special) chars"]`.

### 2026-02-28 — Don't wrap single autogenerated sidebar in a category
**Issue**: Each section had a redundant category wrapper (e.g., "Prompting" category under "MASTERING AI PROMPTS" heading) creating unnecessary nesting
**Rule**: When an HTML section heading contains only one `autogenerated` item, place it directly — do not wrap in a `category`. The category adds a redundant collapsible layer.
