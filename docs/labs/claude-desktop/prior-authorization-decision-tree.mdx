# Prior Authorization Decision Tree

Navigate the complex prior authorization process with an interactive decision tree that guides clinical and operations staff through insurance-specific workflows. The tool helps determine when prior auth is needed, which forms to submit, and how to handle denials -- reducing delays in patient care.

---

## Requirements

The lab brief outlines a decision tree that covers multiple insurance scenarios, prior authorization triggers, and appeal pathways. It should guide users step-by-step through the authorization process with clear branching logic based on insurance type, medication category, and patient history.

![Prior Authorization Decision Tree requirements](/img/labs/claude-desktop/prior-authorization-decision-tree-requirements.png)

---

## Result

The artifact presents an interactive workflow interface with role-based views, compliance checkpoints, and step-by-step task progression. Users navigate through the authorization process with clear guidance on documentation requirements, submission timelines, and follow-up actions.

## How to Recreate This

1. Document your current prior authorization workflows and common insurance scenarios
2. Open [Claude](https://claude.ai) and start a new conversation
3. Describe the decision points and ask for an interactive decision tree with branching logic
4. Iterate -- add insurance-specific pathways, appeal templates, or timeline tracking

:::tip
Decision trees work well for complex operational processes with many conditional branches. Providing real examples of your authorization scenarios helps Claude build accurate branching logic. See [Automation by Function](/automation/automation-by-function) for more workflow automation patterns.
:::
