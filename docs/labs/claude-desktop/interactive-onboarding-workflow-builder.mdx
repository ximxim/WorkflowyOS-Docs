# Interactive Onboarding Workflow Builder

A role-specific onboarding tracker that walks new hires through pre-boarding, Day 1, first week, first month, and 30-day review milestones -- with built-in telehealth compliance checkpoints for HIPAA training, provincial licensing, and malpractice insurance verification.

---

## Requirements

The lab brief describes a workflow builder that supports four roles (Clinician, Engineer, Operations, Marketing) with tasks organized by onboarding phase. Each task should have deadlines, ownership assignments, and compliance tags. The tool should track overall and per-role progress with a searchable, filterable task list.

![Interactive Onboarding Workflow Builder requirements](/img/labs/claude-desktop/interactive-onboarding-workflow-builder-requirements.png)

---

## Result

The artifact is a dark-themed task management interface with role-based tabs (Clinician, Engineer, Operations, Marketing), each showing completion percentage. Tasks are grouped by phase (Pre-boarding through 30-Day Review) with compliance badges, deadline indicators, and a search bar with a telehealth-specific filter toggle.

## How to Recreate This

1. List the roles you onboard and their key compliance requirements
2. Open [Claude](https://claude.ai) and start a new conversation
3. Describe your onboarding phases and ask for an interactive tracker with compliance checkpoints
4. Iterate -- add custom task categories, deadline calculations, or integration with your HR systems

:::tip
For healthcare organizations, embedding compliance checkpoints directly into onboarding workflows ensures nothing falls through the cracks. See [Patient Data and Compliance](/governance/patient-data-and-compliance) for more on building compliance into your AI workflows.
:::
